# 프론트 웹 개발 순서

## 현재 상태
- 기본 프로젝트 구조 설정 완료
- Tailwind CSS 설치 완료
- 프로젝트 제목만 출력되는 기본 상태

## 개발 순서

### 1단계: 환경 설정 및 Supabase 연동 준비
1. **Supabase 프로젝트 확인**
   - Supabase Dashboard에서 "vision.echoit.co.kr" 프로젝트 확인
   - 프로젝트 URL 및 API 키 확인

2. **환경 변수 설정**
   - `.env` 파일 생성 (프로젝트 루트)
   - `PUBLIC_SUPABASE_URL` 설정
   - `PUBLIC_SUPABASE_ANON_KEY` 설정
   - `.gitignore`에 `.env` 추가 확인

3. **Supabase 클라이언트 패키지 설치**
   - `@supabase/supabase-js` 패키지 설치
   - `src/lib/supabaseClient.js` 파일 생성

4. **Vercel 환경 변수 설정**
   - Vercel Dashboard에서 환경 변수 추가
   - Production, Preview, Development 환경 모두 설정

### 2단계: 데이터베이스 스키마 확인 및 설정
1. **기존 스키마 확인**
   - Supabase Dashboard → SQL Editor에서 기존 테이블 구조 확인
   - vision.echoit.co.kr 프로젝트의 데이터베이스 스키마 파악

2. **필요한 테이블 확인**
   - 사용자 프로필 테이블
   - 코칭 관련 테이블
   - 기타 필요한 테이블들

3. **RLS (Row Level Security) 정책 확인**
   - 각 테이블의 보안 정책 확인
   - 필요시 정책 수정 또는 추가

### 3단계: 인증 시스템 구축
1. **인증 Store 생성**
   - `src/lib/stores/authStore.js` 생성
   - 로그인, 회원가입, 로그아웃 기능 구현
   - 인증 상태 관리 (Svelte 5 runes 사용)

2. **인증 콜백 페이지 생성**
   - `src/routes/auth/callback/+page.svelte` 생성
   - OAuth 로그인 후 리다이렉트 처리

3. **레이아웃에 인증 초기화 추가**
   - `src/routes/+layout.svelte`에 인증 초기화 로직 추가
   - 전역 인증 상태 관리

### 4단계: 기본 레이아웃 및 UI 구조 설계
1. **공통 레이아웃 컴포넌트**
   - 헤더/네비게이션 컴포넌트
   - 푸터 컴포넌트 (필요시)
   - 사이드바 컴포넌트 (필요시)

2. **인증 관련 페이지**
   - 로그인 페이지 (`/login`)
   - 회원가입 페이지 (`/signup`)
   - 비밀번호 재설정 페이지 (필요시)

3. **기본 스타일링**
   - Tailwind CSS를 활용한 디자인 시스템 구축
   - 공통 컴포넌트 스타일 정의

### 5단계: 프로젝트 상수 및 설정 파일 정리
1. **프로젝트 상수 확장**
   - `src/prj/prjConst.js`에 필요한 상수 추가
   - 도메인, API 엔드포인트 등 설정

2. **환경별 설정**
   - 개발/프로덕션 환경 구분
   - API URL 등 환경별 설정

### 6단계: 주요 기능 페이지 개발
1. **대시보드/홈 페이지**
   - 메인 대시보드 레이아웃
   - 사용자 정보 표시
   - 주요 기능 링크

2. **프로필 관리**
   - 프로필 조회/수정 페이지
   - 프로필 이미지 업로드 (Storage 연동)

3. **코칭 관련 기능**
   - 코칭 목록 페이지
   - 코칭 상세 페이지
   - 코칭 생성/수정 페이지 (관리자)

### 7단계: 공통 유틸리티 및 서비스 함수
1. **API 서비스 함수**
   - `src/lib/services/` 디렉토리 생성
   - 각 기능별 서비스 함수 분리

2. **유틸리티 함수**
   - `src/lib/utils/` 디렉토리 생성
   - 공통으로 사용할 유틸리티 함수들

3. **에러 처리**
   - 공통 에러 처리 로직
   - 에러 메시지 표시 컴포넌트

### 8단계: 보안 및 최적화
1. **보안 설정**
   - RLS 정책 최종 확인
   - 인증이 필요한 페이지 보호
   - 권한별 접근 제어

2. **성능 최적화**
   - 이미지 최적화
   - 코드 스플리팅
   - 로딩 상태 처리

3. **SEO 설정**
   - 메타 태그 설정
   - `app.html`에 기본 메타 정보 추가

### 9단계: 테스트 및 배포
1. **로컬 테스트**
   - 모든 기능 동작 확인
   - 다양한 시나리오 테스트

2. **Vercel 배포 확인**
   - 빌드 성공 확인
   - 프로덕션 환경에서 동작 확인
   - 환경 변수 정상 작동 확인

3. **버그 수정 및 개선**
   - 발견된 문제점 수정
   - 사용자 피드백 반영

## 참고사항
- Svelte 5의 새로운 runes 문법 사용 (`$state`, `$derived`, `$props` 등)
- Supabase의 RLS 정책을 통한 데이터 보안
- Tailwind CSS를 활용한 반응형 디자인
- Vercel의 환경 변수 설정을 통한 보안 관리
