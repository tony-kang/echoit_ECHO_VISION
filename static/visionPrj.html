에코비전(EchoVision) 프로젝트 구축 계획

1차 구축 목표

1. 더존 월별 파일 업로드

  • 각 파일에 대해 엑셀 칼럼(조직코드 / 매출코드 / 비용코드) 매칭 기능
  • 각 파일에 대해
    • 파일 업로드, 파일 삭제
    • 엑셀 데이터 입력 / 삭제 기능

2. 조직 구조 기준 데이터 분류 및 집계

  • 환경설정 - 조직코드(84개), 매출코드(110개), 비용코드(107개) 관리
  • 환경설정 - 금액코드 관리
    • 매출 14개, 비용 14개

3. 로그인 및 권한 관리

  • 회원가입
  • 소셜 로그인(Google): 로그인 시 자동 회원가입 처리
  • 환경설정 - 사용자 관리
    • 역할관리 (일반사용자 / 관리자)
    • 사용자별 조직 최상위 코드(Multiple) 지정
    • 접근권한(활성화)

4. 통합 조직 기준 전사/부서별 매출 목표 및 목표 대비 달성 현황, 손익 기본 조회

  • 년도별
    • 조직(최상위 - 하위1 - 하위2) 선택 후 매출항목 / 비용항목에 대한 금액 조회
현재 상태: Site 기본기능 및 주요기능 테스트 완료

향후 확장 방향 (2차 구축 ~)

0. 솔루션 검토

솔루션으로서의 가치를 지니고 확장을 위해 기본적으로 검토될 필요가 있는 항목

  • 추가적인 소셜 로그인(카카오, ...)
  • 민감정보(엑셀의 급여정보 등) 처리 관련
  • 엑셀파일 다운로드 권한(다운로드는 관리자만 가능하도록 처리)
  • 엑셀이 아닌 시스템 화면을 업무 미팅에 최적화시키는 방법
  • 관리자 업무 효율성
개발 속도: 1차에 비해 프로젝트 개발 속도가 느려질 가능성 높음(기존의 코드유효성을 유지하면서 신규기능 추가 개발)

1. 조직별 필요한 매출항목 / 비용항목에 대해 손익 조회

  • 월별, 분기별, 반기별, 년도별 데이터
    • 계획, 예상, 실적
  • 매출 / 비용 Chart, 매출 / 손익 Chart
  • 사용자별 조직코드에 대한 매출, 비용 조회
진행 상태: 기본코드 작성 완료

2. 년도별 모든 조직에 대한 손익 조회

  • 테이블 형태 표시 → 전사 경영계획 활용
    • 달성현황: 보이는 데이터량이 방대하여 표현방법 추가 검토 필요
  • 조직별 년도별 실적 조회 → 각 조직별 매출/비용 관리 및 활용
    • 년도별 실적 비교

3. Database 검토

  • 기능 검토 후 불필요한 데이터는 엑셀 단계에서 저장되지 않도록 할 필요 있음
    • 현재 모든 데이터 저장
      • 엑셀 개수 × 조직코드 개수 = 84 × 2 = 168개씩 저장
      • 각 1개의 데이터 내부에 매출코드 개수만큼의 데이터가 들어 추가로 들어 있음
    • 불필요한 데이터는 줄여서 데이터베이스 크기를 줄일 수 있음
